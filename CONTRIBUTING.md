# Contributing

不具合報告、改善提案、Pull Request を歓迎します。

## Issue を作成する前に

1. AutoHotkey v2 を使用していることを確認してください。
2. 最新の `US-AltIMEv3.ahk` で再現することを確認してください。
3. 他の AutoHotkey スクリプトやキーボードリマップツールを停止して、競合の有無を確認してください。
4. 既存の Issue に同じ内容がないか確認してください。

## 不具合報告に含める情報

- Windows のバージョン
- AutoHotkey のバージョンと 32 bit / 64 bit
- キーボード配列
- 使用している IME
- 再現手順
- 期待する動作と実際の動作
- 同時に起動している入力・リマップ関連ソフトウェア

## 開発と動作確認

変更後は、少なくとも次の操作を確認してください。

- 左 Alt 単押しで IME が OFF になる
- 右 Alt 単押しで IME が ON になる
- `Alt + Tab` が通常どおり動作する
- `Alt + F4` が通常どおり動作する
- Alt を押しながら文字キーを入力しても IME が誤って切り替わらない
- Alt キーを使用する別の AutoHotkey スクリプトと併用しても、可能な範囲で競合しない
- 通常権限と管理者権限のアプリで挙動を確認する

## コーディング方針

- AutoHotkey v2 の構文を使用する
- HWND とポインタ値には `Ptr` を使用する
- 左右 Alt の重複処理を増やさず、共通関数を利用する
- 動作変更がないリファクタリングと、動作変更を伴う修正を分ける
- コメントはコードから読み取れない意図や制約を説明するために使用する

## Pull Request

Pull Request には以下を記載してください。

- 変更の目的
- 変更前と変更後の挙動
- 確認したテスト項目
- 既知の制約や未確認事項

IME や Alt キーの挙動は環境差が大きいため、対象環境を具体的に記載してください。
