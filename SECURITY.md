# Security Policy

## Supported version

セキュリティ修正は `main` ブランチの最新版を対象とします。古いスクリプトや改変版はサポート対象外です。

## Reporting a vulnerability

入力フック、権限昇格、任意コード実行など、公開前に確認すべき問題を発見した場合は、公開 Issue へ詳細を書き込まないでください。

GitHub の Security タブから Private vulnerability report または Security Advisory を利用してください。利用できない場合は、リポジトリ所有者の GitHub プロフィールを通じて連絡してください。

報告には以下を含めてください。

- 問題の概要
- 再現手順
- 影響を受ける環境
- 想定される影響
- 修正案がある場合はその内容

通常のキー入力不具合やアプリとの競合は、Issue テンプレートから報告してください。
